# コンディション (Conditions)

## はじめに

コンディションはアクションを実行できるかどうかを決定するために使用されます。

コンディションは以下の場所で使用できます：
- [スキルメカニクス](mechanics.md)
- [ドロップテーブル](../drops/drop-tables.md)
- スポナー
- [ランダムスポーン](../random-spawns.md)

複数のコンディションを適用する場合、スキルが実行されるためにはすべての条件が満たされる必要があります。

## タイプ

コンディションは4つのタイプに分類できます：

1. **エンティティコンディション**: エンティティの状態をチェックします。
2. **ロケーションコンディション**: 位置の状態をチェックします。
3. **比較コンディション**: 2つの「もの」の間の条件をチェックします。
4. **メタコンディション**: キャスターやターゲットの固有のプロパティに対してチェックしません。

## 使用方法

コンディションはメタスキル内の3つの異なる場所で使用できます：

### Conditions

このセクションのコンディションはキャスターまたはその位置に対してチェックします。

```yaml
  Conditions:
  - health{h=>10}
```
> キャスターが10HP以上あるかチェックします

### TargetConditions

このセクションのコンディションはメタスキルの継承されたターゲットに対してチェックします。

```yaml
  TargetConditions:
  - health{h=>10}
```
> 継承されたターゲットエンティティが10HP以上あるかチェックし、そうでないエンティティはメタスキルに継承されません。

### TriggerConditions

このセクションのコンディションはメタスキルのトリガーに対してチェックします。

```yaml
  TriggerConditions:
  - health{h=>10}
```
> メタスキルのトリガーが10HP以上あるかチェックします

## 範囲値

範囲値は **#to#** または **#-#** の形式を使用します。

```yaml
YourMagnificentSkill:
  Conditions:
    - altitude{a=1-5}
  TargetConditions:
    - distance{d=1to10} true
```

## コンディションアクション

コンディションアクションを使用して、コンディションに基づいて追加のアクションを実行できます。デフォルトのコンディションアクションは **true** です。

| アクション | 説明 |
|---------|------|
| `true` | このコンディションが満たされた場合にスキルが実行されます |
| `false` | このコンディションが満たされた場合にスキルが実行されません |
| `power [multiplier]` | コンディションが満たされた場合にスキルのパワーを乗数倍します |
| `cast [skill]` | コンディションが満たされた場合に追加のスキルをキャストします |
| `castinstead [skill]` | コンディションが満たされた場合に代わりに別のスキルをキャストします |
| `orElseCast [skill]` | コンディションが満たされない場合に代わりに別のスキルをキャストします |

### 例

```yaml
  Conditions:
  - day true
  Skills:
  - message{m="昼間です！"} @self
```
> キャスターのワールドが昼間の場合のみ実行されます

## 複合コンディション

コンディションは**AND**（`&&`）と**OR**（`||`）ブールオペレーターで括弧によってグループ化できます。

```yaml
  Conditions:
  - ((day false || raining true) && onBlock{material=LIME_CONCRETE}) true
```
> 雨が降っているか昼間でない場合、かつキャスターがLIME_CONCRETEブロックの上にいる場合に条件が満たされます
